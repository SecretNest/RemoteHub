Switch Direct Client

Stream Direct, SecretNest.RemoteHub.StreamDirect, is a library contains the Remote Hub client for attaching to local switch instance directly.

Use this library

This library defines the Remote Hub client, named as RemoteHubSwitchDirectT, which implemented IRemoteHubT and IRemoteHubAdapterT, can be attached to an instance of switch directly.

Switch is designed to connecting clients and routing messages. But the program who hosts switch has no direct way to have access to the clients. Though a dedicated client can also be attached to the switch within the same program, it requires one more client to receive the message relayed from it and the medium for message transportation between these 2 clients. Stream Direct is the easy solution for this need.

Switch Direct is a Remote Hub client combined with a user side, which implemented IRemoteHubT and IRemoteHubAdapterT in the same component, can be attached to the switch and handle the user side operations. No need to work in pair, thus no medium is required.

Type parameter

Type parameter of Stream Direct is used for sending and receiving user messages.

Currently, only byte array (byte[] in C#.Net / Byte() in VB.Net) and string are acceptable. Passing other type while creating while cause an exception thrown.

Due to transportation on switch is based on byte array, when string is chosen while constructing adapter, an encoder could be provided. If not, Encoding.Default will be used.

When string is used, the messages sent will be encoded before putting on switch, and the received messages will be decoded before calling the callback.

The type only apply on the user messages. System messages, like those for client discovering, will not be affected.

Callback

A instance of OnMessageReceivedCallbackT need to be prepared for receiving user messages. Type of the delegate is the same as the type parameter passed into the Stream Direct.

The callback instance need to be passed as an argument of the constructor of Stream Direct.
