Stream Client

Stream Client, SecretNest.RemoteHub.Stream, is a library contains the Remote Hub client based on streams.

Use this library

This library defines the Remote Hub client works in pairs connected by streams, named as RemoteHubStream, which implemented IRemoteHubOverStream and IRemoteHubT.

In this client, a Stream Adapter will be created to handle the operations on streams.

Type parameter

Type parameter of Stream Client will be passed to Stream Adapter for sending and receiving user messages.

Currently, only byte array (byte[] in C#.Net / Byte() in VB.Net) and string are acceptable. Passing other type while creating while cause an exception thrown.

Due to transportation on streams is based on byte array, when string is chosen while constructing adapter, an encoder could be provided. If not, Encoding.Default will be used.

Using different type parameters or encoders on the both sides of the clients created on the same streams is acceptable. For example, if one side is set as string with Encoding.Default and another is using byte array, the text sent from the 1st one will be received by the 2nd as the encoded output of Encoding.Default.

The type only apply on the user messages. System messages, like those for client discovering, will not be affected.

Callback

A instance of OnMessageReceivedCallbackT need to be prepared for receiving user messages. Type of the delegate is the same as the type parameter passed into the Stream Client.

The callback instance need to be passed as an argument of the constructor of Stream Client.
