Switch

Switch, SecretNest.RemoteHub.Switch, is a library contains a component for connecting adapters and routing messages.

Use this library

This library defines the Switch, named as RemoteHubSwitch.

Switch is designed to connecting clients and routing messages.

Adapters can be attached to switch.

Switches can be cascaded. When using cascading, be careful not to build a shortcut or ring networking, which will fail the switch from working.

Type parameter of Adapters

Switch supports the adapter with byte array (byte[] in C#.Net / Byte() in VB.Net) as the type parameter only.

Switch will not try to decode the user messages, thus it's no harm to leave the message as it is from low level transportation. By using byte array, all user messages will be routed to the target with no change.

Redis Adapter and Stream Adapter can accept the mismatch in type parameter if no user message need to be dealt. When they are connected to the medium which connecting adapters in other kind of type parameter, these adapters should be nothing wrong while creating with byte array as the type parameter for attaching to a switch instance.

Switch Direct can handle the string encoding and decoding automatically. No matter the type parameter chosen, it can always be attached to a switch instance.
