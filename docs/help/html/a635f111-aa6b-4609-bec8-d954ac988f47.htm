Stream Adapter

Redis Adapter, SecretNest.RemoteHub.RedisAdapter, is a library contains adapter of Remote Hub based on Redis database.

Use this library

This library defines the adapter works in pairs connected by streams, named as StreamAdapterT, which implemented IRemoteHubStreamAdapter and IRemoteHubAdapterT.

Creating an instance of this adapter and attaching it to one switch will link the other adapter which connected to the same streams to the switch. This will allow all clients from the other side and the switch to discover each other and transport message among them.

Stream Client, SecretNest.RemoteHub.Stream, will contains an instance of this adapter. But the adapter is constructed and controlled by client. It's not necessary for user to create the instance of adapter for the client.

Type parameter

Type parameter of Stream Adapter is used for sending and receiving user messages.

Currently, only byte array (byte[] in C#.Net / Byte() in VB.Net) and string are acceptable. Passing other type while creating while cause an exception thrown.

Due to transportation on streams is based on byte array, when string is chosen while constructing adapter, an encoder could be provided. If not, Encoding.Default will be used.

Using different type parameters or encoders on the both sides of the adapters created on the same streams is acceptable. For example, if one side is set as string with Encoding.Default and another is using byte array, the text sent from the 1st one will be received by the 2nd as the encoded output of Encoding.Default.

The type is applied on the user messages only. System messages, like those for client discovering, will not be affected.

Callback

One or more instance of OnMessageReceivedCallbackT need to be prepared for receiving user messages. Type of the delegate is the same as the type parameter passed into the Stream Adapter.

The callback instance can to be passed as an argument of the constructor of Stream Adapter, or set it as null for leaving it not initialized.

Use method AddOnMessageReceivedCallback and RemoveOnMessageReceivedCallback to add or remove the callback instance after the adapter constructed.

Adapter

Adapter is a component for Remote Hub which handle the communication on the particular medium.

Adapter instance will:

  • Announce the local client to the medium;

  • Listen to the changes of the remote clients from the medium;

  • Send message to the medium;

  • Receive message, which the target is one of the registered local clients, from the medium; and,

  • Maintain virtual host table.

As Stream Adapter, the medium is a pair of streams, one for input and another for output, or one stream that support duplex communication. Stream Adapter works in pair by using the streams, sending messages from one side to another.
