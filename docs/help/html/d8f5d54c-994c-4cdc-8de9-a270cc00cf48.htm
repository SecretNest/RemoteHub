Event support

Service object with event need to be taken care due to stateful instance.

Stateful service object

When any event is handled remotely, the service object will be turned into stateful instance due to handler removal required when client disconnected.

Although while the proxy client disposing, a message will be sent to the server side to remove the handler registered. When there is any exception related to network or other reason, server side may never receive the message of client disconnecting. To avoid messages sent to a disconnected clients, user code on the server side should monitor the RemoteClientRemoved or RemoteClientRemoved. When a remote client is removed, calls OnProxiesOfSiteDisposed on Remote Agency Manager.

Network stability is not guaranteed always. Client removed event may be raised incorrectly while the link to the client is not stable. Removal the handler of the client may stop the client receiving further event.

And, while using virtual host function, due to the limitation introduced from stateful instance, user code needs to keep the real client id from the first resolving to make sure the same service object is chosen all the time for one client.
