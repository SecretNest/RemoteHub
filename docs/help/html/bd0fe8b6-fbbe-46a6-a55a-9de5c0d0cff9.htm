Welcome to the Remote Hub

Remote Hub is a network communicating solution for Remote Agency.

Remote Hub can be used to connect sites of Remote Agency for network transportation. It can also be used to connect other kinds of data flow with text or byte array format.

Getting Started

Welcome to Remote Hub.

Remote Hub is a solution for Remote Agency, connecting sites by using Redis database and / or network streams.

Remote Hub can also be used dedicatedly, transporting messages among clients. One thing need to be mentioned is, the source (sender) of the message will not be transport to the target. The receiver can only get the message body and the receiver id specified by the sender, but the sender id is not included unless it is a field included in your message body.

The things user should handle:

  • Redis server instance, if you want to transfer messages using Redis publish-subscribe service;

  • Streams for transporting messages, if you want to transfer message using streams;

  • Component for generating message to sent and processing message received, e.g. Remote Agency instance;

  • Reference to Remote Hub.

The things Remote Hub cares:

  • Maintaining state of each client by monitoring clients adding, changing and removing operations;

  • Connecting clients to the same logical network and routing messages among them;

  • Supporting virtual host with load balancing.
